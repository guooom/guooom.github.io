pikichu暴力破解

发表于2022-05-15|更新于2022-05-15
|阅读量:

一、基于表单的简单爆破
1.由于不存在验证码直接利用bp进行暴力破解。抓包，将包丢给Intruder进行暴力 破解，Positions中选着攻击类型（四种），然后选着清除所有的标志符，在要修改的地 方加上标识符

⦁ 进入Payloads界面，导入字典，确定爆破元素

3.进行爆破攻击

⦁ 验证码绕过(on server)
随便输入一个密码和账号，直接进行抓包测试

显示用户名或密码错误，在bp里更改密码交到重发器，换密码后重发来试试，发现报错还是用户名或密码错误，这说明这个验证码可以复用。
这时，我们选着用户，然后对用户密码进行暴力破解，观看破解后的返回字符长来判断那个正确。（因为正确的和错误的返回页面不一样，故字符串长度不一样）

三、验证码绕过(on client)

验证码错误后发现有弹窗，这说明很可能存在前端验证。
然后输入正确验证码进行抓包，直接就绕过了前端的认证
与服务器认证一样，剩下的直接进行爆破就可以了

四、token防爆破
①什么是token - 简书

（1）Token的引入：Token是在客户端频繁向服务端请求数据，服务端频繁的去数据库 查询用户名和密码并进行对比，判断用户名和密码正确与否，并作出相应提示，在这样 的背景下，Token便应运而生。

（2）Token的定义：Token是服务端生成的一串字符串，以作客户端进行请求的一个令 牌，当第一次登录后，服务器生成一个Token便将此Token返回给客户端，以后客户端 只需带上这个Token前来请求数据即可，无需再次带上用户名和密码。

（3）使用Token的目的：Token的目的是为了减轻服务器的压力，减少频繁的查询数据 库，使服务器更加健壮
没有这个token值得话，无法去请求到这个数据，但是这个靶场有token回显
过程：
1.抓包，然后我们可以看见他的token值

然后观察页面源代码，发现前端页面里面存在token的值，所以我们可以利用每次回显 的token值进行爆破，获取每一次的token值，每次放在token位置进行重发。
2.利用回显的token值进行爆破
选择pitchfork类型（pitchfork：每个变量对应一个相应的字典，去每个字典对应值。），设置变量
对变量1的payload进行设置

3.调整option获取源代码中的回显值
改为1个线程

设置重定向

添加Grep-extract

选取token值

4.对变量2进行设置（token）

攻击成功

⦁ 攻击类型有四种：
总结：
Attack Type Payloads set（字典数） 结果
Sniper 1 一个参数保持初始值不变，另一个遍历字典
Battering ram 1 两个参数同时遍历同一个字典
Pitchfork n （取决于参数选几个） 两个参数同时遍历两个不同的字典
Cluster bomb n （取决于参数选几个） 两个字典的笛卡尔积遍历

1.Sniper:可以理解为一个一个爆破，

选择sniper，然后标记name和pwd 两个参数

payload set只能选一个，也就是字典只能设置一个，然后用字典替换选择的参数

2.Battering ram(破城槌):payload set 依然只能选一个，也就是字典只能设置一个

结果就是同时将name和pwd替换为字典内容遍历一遍，name和pwd都一样。

⦁ Pitchfork(音叉):

payload set 可以选2个，也就是字典可以设置2个

4.Clusterbomb(集束炸弹):一般用于两个

payload set 可以选2个，也就是字典可以设置2个

结果就是字典1和字典2的笛卡尔积。也就是将name替换为字典1中的参 数1，然后pwd遍历字典2，然后将name替换为字典1中的参数2，然后pwd 遍历字典2……
